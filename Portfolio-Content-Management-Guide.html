üéØ System Overview

Your portfolio website uses a centralized content management system where all blog posts and projects are stored in data files. This approach makes it easy to add, edit, or remove content without touching complex UI code.

Key Benefits:
  • ‚úÖ No need to edit complex React components
  • ‚úÖ Content is separated from presentation logic
  • ‚úÖ Easy to backup and version control content
  • ‚úÖ Consistent formatting and structure
  • ‚úÖ TypeScript ensures data integrity

Core Data Files

File Purpose Contains
src/data/blog-posts.ts Blog metadata Titles, summaries, dates, tags, categories
src/data/blog-content.ts Blog content Full article content in Markdown format
src/data/projects.ts Project data Project details, tech stacks, metrics, links

üìù Blog Management

Adding a New Blog Post

Follow these detailed steps to add a new blog post to your website:

1Plan Your Content

Before writing, prepare:

  • Title: Clear, descriptive, SEO-friendly
  • Category: Choose from Learning, Analytics, Web3, Tutorial, or AI
  • Tags: 3-5 relevant keywords for searchability
  • Slug: URL-friendly version (lowercase, hyphens)
  • Summary: 1-2 compelling sentences

2Add Blog Metadata

Open src/data/blog-posts.ts and add your post to the blogPosts array:

export const blogPosts: BlogPost[] = [ // ... existing posts ... { title: "Advanced SQL Techniques for Blockchain Analysis", summary: "Learn advanced SQL patterns and techniques specifically designed for analyzing blockchain data. From window functions to complex joins, master the queries that power Web3 analytics.", date: "Jan 15, 2025", // Format: "MMM DD, YYYY" readTime: "12 min read", // Estimate: ~100-150 words per minute tags: ["SQL", "Blockchain", "Analytics", "Advanced", "Tutorial"], slug: "advanced-sql-blockchain-analysis", // Must be unique! category: "Tutorial", // Must match blogCategories featured: true, // Set to true for homepage feature author: { name: "Data Analyst" } } ]
‚ö†Ô∏è Important Notes:
  • Slug must be unique - Check existing slugs first
  • Category must exist - Use only: "Learning", "Analytics", "Web3", "Tutorial", "AI"
  • Date format matters - Use "MMM DD, YYYY" (e.g., "Jan 15, 2025")
  • Comma placement - Don't forget commas between objects

3Add Blog Content

Open src/data/blog-content.ts and add your full article content:

  1. Save both files

Editing Existing Blog Posts

To Update Metadata:

  1. Find your post in src/data/blog-posts.ts
  2. Update the relevant fields (title, summary, tags, etc.)
  3. Save the file

To Update Content:

  1. Find your post slug in src/data/blog-content.ts
  2. Update the Markdown content
  3. Save the file

Deleting Blog Posts

‚ö†Ô∏è Before Deleting:

Consider the SEO impact of removing content. You might want to redirect old URLs or update content instead of deleting.

  1. Remove the post object from src/data/blog-posts.ts
  2. Remove the corresponding entry from src/data/blog-content.ts
  3. Test to ensure no broken links exist

üîß Troubleshooting

Common Issues and Solutions

Blog Post Not Showing Up

Possible Causes:
  • Slug is not unique (duplicate of existing post)
  • Category doesn't match available categories
  • Missing required fields in metadata
  • Syntax error in TypeScript files
  • Content not added to blog-content.ts
Solutions:
  1. Check browser console for errors
  2. Verify all required fields are filled
  3. Ensure slug is unique and matches between files
  4. Validate category against available options
  5. Check for missing commas or brackets
  6. Run npm run build to check for TypeScript errors

Project Card Not Displaying Correctly

Possible Causes:
  • Status not typed as const (missing "as const")
  • Category doesn't exist
  • Malformed metrics object
  • Missing required fields

Build Errors

‚≠ê Best Practices

Content Strategy

üìù Writing Tips

  • Start with value - What will readers gain?
  • Use active voice - More engaging and direct
  • Include examples - Code, screenshots, real data
  • Structure clearly - Headers, bullets, short paragraphs
  • End with action - Next steps, questions, calls to action

SEO Optimization

  • Keywords in title - Include terms people search for
  • Descriptive URLs - Slugs should be readable
  • Meta descriptions - Summary serves as meta description
  • Internal linking - Link between related posts
  • Tag consistency - Use consistent tag naming

Technical Best Practices

  • Version control - Commit content changes regularly
  • Backup data - Keep copies of your content files
  • Test locally - Always preview before publishing
  • Monitor performance - Check loading times
  • Mobile responsive - Test on different devices

Content Maintenance

  • Regular updates - Keep project statuses current
  • Link checking - Verify external links work
  • Content freshness - Update outdated information
  • Performance metrics - Update project metrics regularly
  • Categorization - Review and refine categories as needed

Publishing Schedule

üí° Recommended Cadence:
  • Blog posts: 1-2 per month minimum
  • Project updates: As completed or milestones reached
  • Content review: Quarterly cleanup and updates
  • Analytics review: Monthly performance check
